Peek

A small server for pages that need review

Upload an HTML file. Share a link. Collect feedback directly on the page.

Peek is a self-hosted tool for reviewing generated reports, prototypes, build artifacts, and one-off HTML pages inside a company.

The basic motion

One command turns a local file into a review link.

Peek keeps the workflow deliberately plain: upload from the CLI or dashboard, choose the visibility, send the URL, and let reviewers comment in context.

terminal peek upload 

      


      

    


    

      
What the server does

      
Peek is not a static host. It is a review layer.


      

        

          
1

          

            Stores the artifact
            Uploaded HTML bytes live in file storage or an S3-compatible bucket. Metadata lives in SQLite.
          

        

        

          
2

          

            Wraps it in a safe viewer
            The uploaded page runs in a sandboxed iframe, separate from Peek’s own sessions and review UI.
          

        

        

          
3

          

            Adds review tools
            Reviewers can leave page comments, text-anchored comments, or element-pinned comments.
          

        

        

          
4

          

            Leaves an audit trail
            Owners can inspect visits, read comments, export upload data, and manage visibility later.
          

        

      

    


    

      
The review surface

Instead of losing comments in chat threads, Peek lets reviewers point at the rendered page itself.

Operational shape

      
Small enough to run. Serious enough to trust.

      
Peek ships as a static Go server binary with pure-Go SQLite. It includes first-run admin setup, OAuth options, API tokens, password-protected uploads, private uploads, retention, quotas, health checks, metrics, and backups.


      

        

          0
          server binary
        

        

          0
          visibility modes
        

        

          0
          CGO required
        

      


      
Public link, password gate, or private account-only access. Google and GitHub OAuth can be enabled for company login. Uploaded HTML remains sandboxed in every mode.

    


    

      
The point

      
HTML deserves a review workflow.

      
Peek gives generated pages a place to live, a link to share, and a way for people to respond without leaving the page.


      

        

          quick start
          localhost:7700
        

        
peekd --addr :7700 --data ./data --base-url http://localhost:7700
peek login --host http://localhost:7700
peek upload demo.html --visibility public
